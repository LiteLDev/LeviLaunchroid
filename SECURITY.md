# Security Policy

Thank you for helping improve the security of LeviLaunchroid.

This document describes the process for reporting security vulnerabilities. It does not define guaranteed response times, support lifecycles, or disclosure timelines unless explicitly stated.

Please **do not** create a public GitHub issue or discuss potential vulnerabilities in public community channels.

When submitting a report, include as much information as possible, such as:

- Description of the vulnerability
- Steps to reproduce
- Affected version(s), if known
- Expected and actual behavior
- Proof of concept, if applicable
- Relevant logs or screenshots

Please also follow the project's [Code of Conduct](CODE_OF_CONDUCT.md) throughout the reporting and disclosure process.

## Responsible Disclosure

To help protect users, avoid publicly disclosing a vulnerability before it has been reviewed. Public disclosure may increase risk for users before a fix or mitigation is available.

## Scope

Examples of security-related issues include, but are not limited to:

- Arbitrary code execution
- Privilege escalation
- Authentication or authorization bypass
- Information disclosure
- Unsafe handling of imported APKs or native modules
- Path traversal during file operations

General bugs, feature requests, installation problems, and documentation improvements should continue to be reported through GitHub Issues.

## Policy Notes

At the time of writing, this project does not publish:

- a guaranteed response time,
- a guaranteed remediation timeline,
- a formal supported-version policy,
- a vulnerability disclosure schedule.

Unless these policies are defined in future revisions of this document, no specific guarantees should be assumed.

This policy may be updated as the project's security practices evolve.
